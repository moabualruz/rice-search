# Historical Documentation

This directory contains historical fix documentation and migration plans from the development of Rice Search. These files document specific issues that were resolved and system changes that were implemented.

## Purpose

These files serve as:
- **Historical Record** - Track of issues fixed and improvements made
- **Reference Material** - Understanding of why certain decisions were made
- **Troubleshooting Context** - Background for similar future issues

## Files

### Bug Fixes & Improvements
- **DEDUPLICATION_FIX.md** - Fix for duplicate file entries in search results
- **DIMENSION_FIX.md** - Resolution of vector dimension mismatch (768 vs 2560)
- **SCORE_NORMALIZATION_FIX.md** - Search score normalization improvements
- **FILE_PATH_INDEXING_ENHANCEMENT.md** - Enhancement to file path searchability
- **UI_IMPROVEMENTS.md** - Frontend improvements for file path display
- **SEARCH_LLM_RERANKER_FIXES.md** - LLM reranker bug fixes

### System Changes & Migrations
- **DEPENDENCY_UPDATES.md** - Major dependency updates
- **UNIFIED_INFERENCE_MIGRATION.md** - Migration to unified inference service
- **SGLANG_MIGRATION_PLAN.md** - SGLang integration planning
- **SETTINGS_SYSTEM.md** - Settings management system design
- **SETTINGS_AUTO_PERSIST.md** - Auto-persistence feature for settings
- **SETTINGS_UI_SUMMARY.md** - Settings UI implementation summary
- **QUICK_CONFIG_MODERNIZATION.md** - Configuration system modernization

### Optimization & Recommendations
- **MODEL_OPTIMIZATION_RECOMMENDATIONS.md** - Model performance optimization recommendations

## Note

These files are **historical documentation** and may reference code or configurations that have since changed. For current documentation, see the main [docs/](../) directory.
